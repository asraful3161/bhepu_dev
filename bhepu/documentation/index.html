Carlisting

Auto and Car Listing Multi-Vendor Directory CMS

Author: xicia
Create Date: 19-12-2017
Support Email: jbbr.1990@gmail.com

Overview - Top

Carlisting is a responsive car listing directory Content Management System (CMS). Sellers can register and manage items easily through making a payment through PayPal. In this CMS, seller will give their car advertisements (to sale new and old cars) through paying to the website owner. Visitors will come and see the cars and contact the sellers through their email, phone or message forms for each individual cars.

This CMS has a nice and attractive front end and back end interface that are really awesome and eyecatching. The back end has a lot of nice and maintainable features that are needed for a modern and professional website to handle the contents easily by client. Almost all kinds of changes are possible to do using the admin panel without having any kind or programming language knowledge.

This CMS is built using the procedural php (with PDO feature) and without any framework. For this reason, it can be customized easily by most of the developers.

This CMS is protected against SQL injection, XSS and other attack and your website will be safe. Also in all pages, admin level security and invalid page url submission checking were done.

Requirements - Top

Before starting to install our item, make sure you fulfill the following requirements:

  1. For Local Server installation, you need to have XAMPP, WAMP or any apache server with PHP and MySQL.
  2. Mod Rewrite will have to be enabled if you want to get SEO friendly URL. Examples are given below:

    If Mod Rewrite is not enabled, a page url will be like: 
    http://www.yourwebsite.com/page.php?slug=about-us
    If Mod Rewrite is enabled, a page url will be like: 
    http://www.yourwebsite.com/page/about-us
  3. You have use any live LINUX server to use our script without problem. We do not give guarantee to use our script in local server without problem (it will depend totally up on you.)
  4. Required PHP version in server >= 5.4.
  5. PDO must be enabled in the server.

Installation - Top

In order to install our script, follow the steps below:

  1. Download the main zip file from Codecanyon.
  2. Now unzip it and into to upload folder, you will get a zip file cms.zip.
  3. Upload the cms.zip file into your server using FTP or Cpanel. Extract this and use. You can use our php script into root domain or subdomain.
  4. Create mysql database using cpanel or your hosting provider's system. Then import the xicia_carlisting.sql file (you will get it in database folder) into your created database.

Configuration - Top

The configuration steps are very important. Please follow the instruction carefully, otherwise the script will not work well.

  1. Go to the cms folder that you extracted.

  2. Then go to admin folder and open up the config.php file with any text editor (for example: Notepad++).

  3. In this config.php file, you will get some important configuration settings. You must have to edit those properly.
    • Time Zone: Write your proper timezone here
    • Host Name: This is your database host name or IP address
    • Database Name: Create a database name in your host (localhost or webhost) and put name here
    • Database Username: Username of the database you created
    • Database Password: Password of the database you created
    • BASE URL: This is the main website URL where you are trying to run our script. Do not forget to put the trail slash in the URL, otherwise the script will not work.

  4. Also make sure that your .htaccess contains the real data and not empty. Sometimes, during upload web server makes the files hidden and so .htaccess file can not be uploaded. So you must have to be sure that it is in the root of your server.

Car Section - Top

Though cars will be added mainly by seller, you need to make settings before seller can add a car. We are describing what settings do you need to make in below:

Pending Cars

When a seller will add a new car from seller dashboard, the car will be pending and come in this section. Admin can view the car and then approve it. If admin wants, he can also delete this car (if it violates any of the company rules)

Approved Cars

Admin can see all the approved cars from here. He can delete an existing car if needed. But admin can not edit a car information.

Car Category

Admin can add, edit and delete car categories from this section. But one important thing here is that if a car category is used for a car, admin can not delete that car category until he or seller removes all cars under that car category from database.

Brand

Admin can add, edit and delete brand from this section. But one important thing here is that if a brand is used for a car, admin can not delete that brand until he or seller removes all cars under that brand from database.

Model

Admin can add, edit and delete model from this section. But one important thing here is that if a model is used for a car, admin can not delete that model until he or seller removes all cars under that model from database.

Body Type

Admin can add, edit and delete body type from this section. But one important thing here is that if a body type is used for a car, admin can not delete that body type until he or seller removes all cars under that body type from database.

Fuel Type

Admin can add, edit and delete fuel type from this section. But one important thing here is that if a fuel type is used for a car, admin can not delete that fuel type until he or seller removes all cars under that fuel type from database.

Transmission

Admin can add, edit and delete transmission type from this section. But one important thing here is that if a transmission type is used for a car, admin can not delete that transmission type until he or seller removes all cars under that transmission type from database.
